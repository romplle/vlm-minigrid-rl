# MiniGrid Navigation NanoVLM (SFT + GRPO)

## Описание проекта

В этом проекте обучается мультимодальная модель для навигации в среде MiniGrid (MiniGrid-Empty-8x8-v0).
Задача агента — дойти до зелёной клетки цели как можно быстрее, имея частичное наблюдение (7x7 RGB изображение).

Обучение проводится в два этапа:

1. **SFT (Supervised Fine-Tuning)** на экспертных траекториях
2. **GRPO (Group Relative Policy Optimization)** для дообучения через RL

## Генерация экспертных траекторий

Для создания экспертных траекторий использовался алгоритм **BFS (Breadth-First Search)**, который находит кратчайший путь до цели.

Это позволяет:

* получать оптимальные действия,
* формировать корректные пары (состояние → действие),
* обучать модель на “идеальном” поведении.

В проекте используется **NanoVLM v0.1**:

## SFT обучение

Я экспериментировал с:

* различными промптами,
* разными форматами входа,
* разными вариантами улучшения SFT.

### Проблема дисбаланса действий

Большинство действий — `forward`.
Редкие действия — `turn_left` и `turn_right`.

Из-за этого:

* модель склонна переобучаться на `forward`,
* хуже предсказывает повороты,
* хуже работает в ситуациях, когда цель не видна.

Возможные улучшения:

* увеличить датасет,
* сделать балансировку действий,
* усилить примеры с `left/right`.

## Результаты после SFT

Точность модели после SFT: **примерно 60–70%**.

Однако это может быть завышено, так как:

* большинство действий — `forward`,
* редкие действия предсказываются хуже.

То есть accuracy не полностью отражает реальное качество политики.

## GRPO обучение

После SFT модель дообучалась методом GRPO.

После GRPO:

* Пиковая точность достигала **82%**

Однако:

* это требует дополнительной проверки,
* желательно использовать другие метрики (успешность эпизода, длина траектории и т.д.).

## Логирование

Во время обучения использовался **Weights & Biases (wandb)**:

* сохранены все графики,
* сохранены логи,
* доступны кривые обучения.

В некоторых регионах доступ к сервису может быть ограничен.
В README приведена краткая версия результатов. Подробные графики приложены в PDF-отчёте.

## Запуск проекта

1. Клонируйте репозиторий: `git clone https://github.com/romplle/vlm-minigrid-rl`.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Скачайте релиз NanoVLM: https://github.com/huggingface/nanoVLM/releases/tag/v0.1. Поместите скачанную папку в директорию проекта и переименуйте её в nanoVLM.

#### Быстрый тест модели:

4. В репозитории уже находятся подготовленный датасет и чекпоинты моделей, поэтому можно сразу попробовать работу модели. Для этого откройте и запустите ноутбук: `quick_test.ipynb`.

#### Обучение с нуля:

4. Удалите папку dataset.
5. Сгенерируйте датасет экспертных траекторий: `python dataset_generation.py`.
6. Если wandb недоступен, можно отключить логирование (`WANDB_MODE=disabled`) или использовать локальный режим (`WANDB_MODE=offline`).
7. Запустите обучение SFT: `python sft.py`.
8. После завершения SFT запустите RL-дообучение через GRPO: `python grpo.py`.
9. Во время обучения автоматически сохраняются чекпоинты моделей и логи экспериментов (в том числе через wandb).

## Итог

Мне удалось:

* обучить модель через SFT,
* улучшить её через GRPO,
* получить точность до 82%,
* сохранить полные логи экспериментов.

Проект показывает, что даже небольшая мультимодальная модель может обучаться навигации в MiniGrid при правильной генерации экспертных траекторий и использовании RL-донастройки.
